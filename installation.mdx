title: "Installation"
description: "Install COLMAP on Linux, macOS, Windows, or via Docker — using pre-built binaries, package managers, conda, or from source."
---

You can install COLMAP by downloading a pre-built binary, using a package manager, pulling a Docker image, or building from source. Pre-built binaries and other resources are available at [github.com/colmap/colmap/releases](https://github.com/colmap/colmap/releases).

<Note>
  COLMAP packages in the default Linux/Unix/BSD repositories do not include CUDA support. To enable GPU-accelerated dense reconstruction, build from source as described below.
</Note>

## Pre-built binaries

<Tabs>
  <Tab title="Windows">
    Download the pre-built Windows binaries from the [releases page](https://github.com/colmap/colmap/releases). The archive contains both the GUI and command-line interface executables.

    To start the COLMAP GUI, double-click `COLMAP.bat` or run it from Command Prompt or PowerShell. The batch script automatically sets the required library paths.

    To list available commands:

    ```bat
    COLMAP.bat -h
    ```

    <Note>
      On first launch, Windows Defender may show a security warning because the binaries are not officially signed. The binaries are built automatically by GitHub Actions CI. If you do not trust them, build from source instead.
    </Note>
  </Tab>

  <Tab title="Linux / Unix / BSD">
    System packages for many Linux, Unix, and BSD distributions are tracked at [repology.org](https://repology.org/metapackage/colmap/versions).

    <Warning>
      Packages in default repositories do not include CUDA support. For GPU-accelerated dense reconstruction, build from source.
    </Warning>
  </Tab>

  <Tab title="macOS (Homebrew)">
    [Homebrew](https://brew.sh) provides a COLMAP formula with pre-compiled binaries:

    ```bash
    brew install colmap
    ```

    Verify the installation:

    ```bash
    colmap -h
    colmap gui
    ```
  </Tab>

  <Tab title="Conda">
    Install the conda-forge package:

    ```bash
    conda install colmap
    ```

    Packages are available at [anaconda.org/conda-forge/colmap](https://anaconda.org/conda-forge/colmap).
  </Tab>

  <Tab title="Docker">
    Pre-built Docker images with CUDA support are available on Docker Hub:

    ```bash
    docker pull colmap/colmap:latest
    ```

    See the [Docker section](#docker) below for full usage instructions.
  </Tab>
</Tabs>

## Docker

COLMAP provides a pre-built Docker image with CUDA support. The included `run.sh` script handles GPU detection automatically.

**Requirements:**
- Docker 19.03 or newer
- NVIDIA GPU with drivers installed (for dense reconstruction)

<Steps>
  <Step title="Install Docker and the NVIDIA toolkit">
    Verify Docker is installed:

    ```bash
    docker --version
    ```

    Install the NVIDIA Container Toolkit on your host machine. Helper scripts are provided in the `docker/` directory of the repository:

    ```bash
    # Ubuntu
    ./docker/setup-ubuntu.sh

    # CentOS
    ./docker/setup-centos.sh
    ```
  </Step>

  <Step title="Run the container">
    Pass the full local path to your working directory (the folder containing your images):

    ```bash
    ./docker/run.sh /path/to/your/working/directory
    ```

    The script automatically detects whether a GPU is available and configures the container accordingly. It mounts the specified directory to `/working` inside the container and opens an interactive shell.
  </Step>

  <Step title="Run COLMAP inside the container">
    Once inside the container, run COLMAP commands against your mounted data:

    ```bash
    colmap automatic_reconstructor \
        --image_path ./images \
        --workspace_path .
    ```

    To launch the GUI instead:

    ```bash
    ./docker/run-gui.sh /path/to/your/working/directory
    ```
  </Step>
</Steps>

### Build the Docker image from source

If you want to build the image yourself rather than using the pre-built one from Docker Hub:

```bash
# From the repository root
./docker/build.sh
./docker/run.sh /path/to/your/working/directory
```

To target a specific CUDA compute architecture, pass a `--build-arg`:

```bash
docker build . -f docker/Dockerfile -t colmap:latest --build-arg CUDA_ARCHITECTURES=75
```

The Dockerfile uses a two-stage build: a `builder` stage that compiles COLMAP and a minimal `runtime` stage that contains only the runtime dependencies, keeping the final image small.

## Build from source

COLMAP builds on Linux, macOS, and Windows. Start by cloning the repository:

```bash
git clone https://github.com/colmap/colmap
```

<Tabs>
  <Tab title="Debian / Ubuntu">
    **Recommended:** CUDA 11.x or newer for GPU support.

    Install dependencies from the default Ubuntu repositories:

    ```bash
    sudo apt-get install \
        git \
        cmake \
        ninja-build \
        build-essential \
        libboost-program-options-dev \
        libboost-graph-dev \
        libboost-system-dev \
        libeigen3-dev \
        libopenimageio-dev \
        openimageio-tools \
        libmetis-dev \
        libgoogle-glog-dev \
        libgtest-dev \
        libgmock-dev \
        libsqlite3-dev \
        libglew-dev \
        qt6-base-dev \
        libqt6opengl6-dev \
        libqt6openglwidgets6 \
        libcgal-dev \
        libceres-dev \
        libsuitesparse-dev \
        libcurl4-openssl-dev \
        libssl-dev \
        libmkl-full-dev

    # Fix issue in Ubuntu's openimageio CMake config.
    sudo mkdir -p /usr/include/opencv4
    ```

    Alternatively, use Qt 5 instead of Qt 6:

    ```bash
    sudo apt-get install qtbase5-dev libqt5opengl5-dev
    ```

    To enable **CUDA support**, install Ubuntu's CUDA package:

    ```bash
    sudo apt-get install -y \
        nvidia-cuda-toolkit \
        nvidia-cuda-toolkit-gcc
    ```

    Or install the latest CUDA directly from NVIDIA's website.

    <Warning>
      On **Ubuntu 22.04**, there is a known issue when compiling with Ubuntu's default CUDA package and GCC. You must compile against GCC 10:

      ```bash
      sudo apt-get install gcc-10 g++-10
      export CC=/usr/bin/gcc-10
      export CXX=/usr/bin/g++-10
      export CUDAHOSTCXX=/usr/bin/g++-10
      ```
    </Warning>

    Configure and build:

    ```bash
    cd colmap
    mkdir build
    cd build
    cmake .. -GNinja -DBLA_VENDOR=Intel10_64lp
    ninja
    sudo ninja install
    ```

    When targeting a specific GPU, pass `-DCMAKE_CUDA_ARCHITECTURES=native` to CMake. Use `"all"` or `"all-major"` to produce a binary that runs on other machines.

    <Tip>
      The `BLA_VENDOR=Intel10_64lp` flag tells CMake to use Intel MKL for BLAS. If you prefer OpenBLAS, install the OpenMP variant to avoid a known OpenMP/OpenBLAS interaction issue.
    </Tip>

    Run COLMAP:

    ```bash
    colmap -h
    colmap gui
    ```
  </Tab>

  <Tab title="macOS">
    Install dependencies with [Homebrew](https://brew.sh):

    ```bash
    brew install \
        cmake \
        ninja \
        boost \
        eigen \
        openimageio \
        curl \
        libomp \
        metis \
        glog \
        googletest \
        ceres-solver \
        suitesparse \
        qt \
        glew \
        cgal \
        sqlite3
    brew link --force libomp
    ```

    Configure and build:

    ```bash
    cd colmap
    mkdir build
    cd build
    cmake .. -GNinja
    ninja
    sudo ninja install
    ```

    <Note>
      If you have Qt 5 and Qt 6 both installed, temporarily force-link your Qt 6 installation before running CMake:

      ```bash
      brew unlink qt && brew link --force qt
      cmake ..
      ```
    </Note>

    Run COLMAP:

    ```bash
    colmap -h
    colmap gui
    ```
  </Tab>

  <Tab title="Windows (VCPKG)">
    **Recommended:** CUDA 11.x or newer, Visual Studio 2019 or newer.

    The recommended approach on Windows is to use [VCPKG](https://github.com/microsoft/vcpkg) to build COLMAP and all its dependencies:

    ```powershell
    git clone https://github.com/microsoft/vcpkg
    cd vcpkg
    .\bootstrap-vcpkg.bat
    .\vcpkg install colmap[cuda,tests]:x64-windows
    ```

    To compile CUDA support for multiple compute architectures:

    ```powershell
    .\vcpkg install colmap[cuda-redist]:x64-windows
    ```

    <Note>
      CUDA must be installed separately using NVIDIA's standard installer before running the VCPKG commands above.
    </Note>
  </Tab>

  <Tab title="VCPKG (cross-platform)">
    COLMAP ships as part of the VCPKG distribution, enabling builds on any supported platform. CUDA must be installed manually first.

    ```bash
    git clone https://github.com/microsoft/vcpkg
    cd vcpkg
    ./bootstrap-vcpkg.sh

    # Linux (basic)
    ./vcpkg install colmap:x64-linux

    # Linux with CUDA and tests
    ./vcpkg install colmap[cuda,tests]:x64-linux

    # Latest development version
    ./vcpkg install colmap:x64-linux --head
    ```

    VCPKG supports other platforms as well: `x64-osx`, `x64-windows`, etc.

    To build from a separate source directory using VCPKG-managed dependencies, first run `./vcpkg integrate install` (on Windows, use PowerShell and `./scripts/shell/enter_vs_dev_shell.ps1`), then:

    ```bash
    cd path/to/colmap
    mkdir build
    cd build
    cmake .. \
        -DCMAKE_TOOLCHAIN_FILE=path/to/vcpkg/scripts/buildsystems/vcpkg.cmake \
        -DCMAKE_BUILD_TYPE=Release
    cmake --build . --config release --target colmap --parallel 24
    ```
  </Tab>

  <Tab title="Anaconda / Mamba">
    Install Miniconda, then create and configure the environment. You can replace `conda` with `mamba` for faster package resolution:

    ```bash
    conda create -n colmap python=3.12

    conda config --set channel_priority strict

        cuda-compiler==12.6.2 \
        cuda-cudart-dev \
        cuda-nvrtc-dev \
        libcurand-dev
    ```

    Clone and build:

    ```bash

    cd colmap
    mkdir build
    cd build
    cmake .. -GNinja
    ninja
    ```

## Building as a library

You can include and link COLMAP in your own C++ projects. After building and installing COLMAP, the following files are placed in your install prefix:

- Headers: `${CMAKE_INSTALL_PREFIX}/include/colmap`
- Libraries: `${CMAKE_INSTALL_PREFIX}/lib/colmap`
- CMake config: `${CMAKE_INSTALL_PREFIX}/share/colmap`

Use `find_package(colmap REQUIRED)` in your `CMakeLists.txt`:

<CodeGroup>

```cmake CMakeLists.txt
cmake_minimum_required(VERSION 3.10)

project(SampleProject)

find_package(colmap REQUIRED)
# or to require a specific version: find_package(colmap 3.4 REQUIRED)

add_executable(hello_world hello_world.cc)
target_link_libraries(hello_world colmap::colmap)
```

```cpp hello_world.cc
#include <cstdlib>
#include <iostream>

#include <colmap/controllers/option_manager.h>
#include <colmap/util/string.h>

int main(int argc, char** argv) {
    colmap::InitializeGlog(argv);

    std::string message;
    colmap::OptionManager options;
    options.AddRequiredOption("message", &message);
    if (!options.Parse(argc, argv)) {
        return EXIT_FAILURE;
    }

    std::cout << colmap::StringPrintf("Hello %s!\n", message.c_str());

    return EXIT_SUCCESS;
}
```

</CodeGroup>

Build and run:

```bash
mkdir build
cd build
export colmap_DIR=${CMAKE_INSTALL_PREFIX}/share/colmap
cmake .. -GNinja
ninja
./hello_world --message "world"
```

A complete example is available under `doc/sample-project` in the repository.

### Shared libraries

By default, COLMAP builds static libraries. To build shared libraries instead:

```bash
cmake .. -GNinja -DBUILD_SHARED_LIBS=ON
```

| Trade-off | Static | Shared |
|---|---|---|
| Incremental linking speed | Slower (relinks all executables) | Faster (relinks only changed library) |
| Disk usage | Higher | Lower (shared on disk and in memory) |
| Runtime performance | Better (LTO/IPO possible) | Slightly lower (no cross-library inlining) |
| Startup cost | None | Minor (dynamic linker overhead) |

<Tip>
  Use shared libraries during development to speed up edit-compile-test cycles. Use static libraries for production or benchmarking.
</Tip>

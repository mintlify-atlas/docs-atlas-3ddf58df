---
title: "Command-line Interface"
description: "Run COLMAP's full reconstruction pipeline from the command line for scripting, automation, and headless server use."
---

The command-line interface provides access to all of COLMAP's functionality. Each capability is implemented as a subcommand of the `colmap` executable.

```bash
colmap [command] [options]
```

<Note>
  On Windows, use `COLMAP.bat` in place of `colmap`. If running from the CMake build folder, the executable is at `./src/colmap/exe/colmap`.
</Note>

## Getting help

List all available commands:

```bash
colmap help
```

Show options for any command:

```bash
colmap <command> -h
```

For example:

```bash
colmap feature_extractor -h
```

## Project configuration files

All options can be provided as command-line arguments or through a `.ini` project file:

```bash
colmap feature_extractor --project_path /path/to/project.ini
```

The `.ini` file stores all parameter settings including database and image paths. You can generate one from the GUI (**File > Save project**), generate it with the `project_generator` command, or write it by hand in a text editor.

Example `.ini` fragment:

```ini
[General]
database_path=/path/to/project/database.db
image_path=/path/to/project/images

[ImageReader]
camera_model=SIMPLE_RADIAL
single_camera=0

[SiftExtraction]
max_num_features=8192
```

## Headless (no display) usage

COLMAP automatically switches to CUDA-based feature extraction and matching when no display is attached. To force CPU mode:

```bash
colmap feature_extractor \
  --database_path database.db \
  --image_path images/ \
  --FeatureExtraction.use_gpu 0

colmap exhaustive_matcher \
  --database_path database.db \
  --FeatureMatching.use_gpu 0
```

## Quick start: automatic reconstruction

The simplest way to run a full sparse and dense reconstruction from a folder of images:

```bash
# The project folder must contain a folder "images" with all the images.
DATASET_PATH=/path/to/project

colmap automatic_reconstructor \
    --workspace_path $DATASET_PATH \
    --image_path $DATASET_PATH/images
```

Key options for `automatic_reconstructor`:

| Option | Values | Description |
|--------|--------|-------------|
| `--quality` | `LOW`, `MEDIUM`, `HIGH`, `EXTREME` | Trade-off between speed and reconstruction quality. |
| `--data_type` | `INDIVIDUAL`, `VIDEO`, `INTERNET` | Tunes settings for the capture scenario. |
| `--feature` | `SIFT`, `ALIKED` | Feature extraction algorithm. |
| `--mapper` | `INCREMENTAL`, `HIERARCHICAL`, `GLOBAL` | SfM pipeline to use. |
| `--mesher` | `POISSON`, `DELAUNAY` | Surface reconstruction method. |

## Step-by-step reconstruction

For full control over the pipeline, run each step individually:

```bash
DATASET_PATH=/path/to/dataset

# 1. Extract features
colmap feature_extractor \
   --database_path $DATASET_PATH/database.db \
   --image_path $DATASET_PATH/images

# 2. Match features
colmap exhaustive_matcher \
   --database_path $DATASET_PATH/database.db

# 3. Sparse reconstruction
mkdir -p $DATASET_PATH/sparse
colmap mapper \
    --database_path $DATASET_PATH/database.db \
    --image_path $DATASET_PATH/images \
    --output_path $DATASET_PATH/sparse

# 4. Undistort images for MVS
mkdir -p $DATASET_PATH/dense
colmap image_undistorter \
    --image_path $DATASET_PATH/images \
    --input_path $DATASET_PATH/sparse/0 \
    --output_path $DATASET_PATH/dense \
    --output_type COLMAP \
    --max_image_size 2000

# 5. Dense stereo
colmap patch_match_stereo \
    --workspace_path $DATASET_PATH/dense \
    --workspace_format COLMAP \
    --PatchMatchStereo.geom_consistency true

# 6. Fuse depth maps
colmap stereo_fusion \
    --workspace_path $DATASET_PATH/dense \
    --workspace_format COLMAP \
    --input_type geometric \
    --output_path $DATASET_PATH/dense/fused.ply

# 7. Surface reconstruction (choose one)
colmap poisson_mesher \
    --input_path $DATASET_PATH/dense/fused.ply \
    --output_path $DATASET_PATH/dense/meshed-poisson.ply

colmap delaunay_mesher \
    --input_path $DATASET_PATH/dense \
    --output_path $DATASET_PATH/dense/meshed-delaunay.ply
```

## Global SfM pipeline

Replace the `mapper` step with `global_mapper` to use the global SfM pipeline (rotation averaging + global positioning). The global mapper requires good focal length priors. Run `view_graph_calibrator` first if reliable intrinsics are not available:

```bash
# Optional but recommended: calibrate intrinsics from the view graph.
# This modifies the database in-place, so work on a copy.
cp $DATASET_PATH/database.db $DATASET_PATH/database_global.db
colmap view_graph_calibrator \
    --database_path $DATASET_PATH/database_global.db

mkdir -p $DATASET_PATH/sparse
colmap global_mapper \
    --database_path $DATASET_PATH/database_global.db \
    --image_path $DATASET_PATH/images \
    --output_path $DATASET_PATH/sparse
```

## All commands

### Feature extraction and import

<AccordionGroup>
  <Accordion title="feature_extractor">
    Detect and describe keypoints in images. Supports SIFT (CPU/GPU) and ALIKED (ONNX-based neural features). Images already processed in a previous run are skipped automatically.

    ```bash
    colmap feature_extractor \
      --database_path database.db \
      --image_path images/ \
      --ImageReader.camera_model SIMPLE_RADIAL \
      --FeatureExtraction.type SIFT \
      --SiftExtraction.max_num_features 8192
    ```
  </Accordion>
  <Accordion title="feature_importer">
    Import pre-computed features from text files located alongside each image. Each `.txt` file must follow the format `NUM_FEATURES 128` on the first line, followed by one feature per line: `X Y SCALE ORIENTATION D_1 ... D_128`.

    ```bash
    colmap feature_importer \
      --database_path database.db \
      --image_path images/
    ```
  </Accordion>
</AccordionGroup>

### Feature matching

<AccordionGroup>
  <Accordion title="exhaustive_matcher">
    Match every image against every other image. Best for datasets with up to a few hundred images. Provides the most complete match graph.

    ```bash
    colmap exhaustive_matcher \
      --database_path database.db
    ```
  </Accordion>
  <Accordion title="sequential_matcher">
    Match consecutively captured images. Built-in loop detection using a vocabulary tree. Image file names must be in sequential order. Recommended for video sequences.

    ```bash
    colmap sequential_matcher \
      --database_path database.db \
      --SequentialMatching.loop_detection 1 \
      --SequentialMatching.vocab_tree_path vocab_tree.bin
    ```
  </Accordion>
  <Accordion title="vocab_tree_matcher">
    Match each image against its visual nearest neighbors using a vocabulary tree with spatial re-ranking. Recommended for large image collections (thousands of images). Requires a pre-trained vocabulary tree (downloadable from https://demuc.de/colmap/).

    ```bash
    colmap vocab_tree_matcher \
      --database_path database.db \
      --VocabTreeMatching.vocab_tree_path vocab_tree.bin
    ```
  </Accordion>
  <Accordion title="spatial_matcher">
    Match images against their spatial nearest neighbors using GPS coordinates from EXIF or manually entered locations. Use this when accurate prior location information is available.

    ```bash
    colmap spatial_matcher \
      --database_path database.db
    ```
  </Accordion>
  <Accordion title="transitive_matcher">
    Use transitivity of existing matches to produce a more complete graph. If A matches B and B matches C, this matcher attempts to directly match A to C.

    ```bash
    colmap transitive_matcher \
      --database_path database.db
    ```
  </Accordion>
  <Accordion title="matches_importer">
    Import pre-computed raw or geometrically verified matches from a text file. Expected format: image pair header followed by zero-based feature index pairs.

    ```bash
    colmap matches_importer \
      --database_path database.db \
      --match_list_path matches.txt \
      --match_type raw
    ```
  </Accordion>
</AccordionGroup>

### Geometric verification

<AccordionGroup>
  <Accordion title="geometric_verifier">
    Run standalone geometric verification on existing feature matches. Estimates two-view geometries (fundamental/essential matrices, homographies) for matched image pairs.

    ```bash
    colmap geometric_verifier \
      --database_path database.db
    ```
  </Accordion>
  <Accordion title="guided_geometric_verifier">
    Run geometric verification guided by an existing sparse reconstruction. Uses known relative camera poses to improve match verification results.

    ```bash
    colmap guided_geometric_verifier \
      --database_path database.db \
      --input_path sparse/0
    ```
  </Accordion>
</AccordionGroup>

### Sparse reconstruction

<AccordionGroup>
  <Accordion title="mapper">
    Incremental sparse 3D reconstruction. Seeds from an initial image pair and grows the model by registering new images and triangulating points. Produces one or more models in the output folder.

    ```bash
    colmap mapper \
      --database_path database.db \
      --image_path images/ \
      --output_path sparse/
    ```
  </Accordion>
  <Accordion title="global_mapper">
    Global SfM pipeline that solves for all camera poses simultaneously using rotation averaging and global positioning. Faster than incremental SfM on large datasets but less robust to outliers. Requires good focal length priors — run `view_graph_calibrator` first if intrinsics are not available.

    ```bash
    colmap global_mapper \
      --database_path database_global.db \
      --image_path images/ \
      --output_path sparse/
    ```
  </Accordion>
  <Accordion title="hierarchical_mapper">
    Hierarchical SfM that partitions the scene into overlapping submodels, reconstructs each independently in parallel, and merges them into a single reconstruction. Run point triangulation and bundle adjustment after this step.

    ```bash
    colmap hierarchical_mapper \
      --database_path database.db \
      --image_path images/ \
      --output_path sparse/
    ```
  </Accordion>
  <Accordion title="pose_prior_mapper">
    Sparse reconstruction using pose priors (e.g., from GPS or IMU) to constrain camera positions during mapping.

    ```bash
    colmap pose_prior_mapper \
      --database_path database.db \
      --image_path images/ \
      --output_path sparse/
    ```
  </Accordion>
  <Accordion title="point_triangulator">
    Triangulate all observations of registered images in an existing model using feature matches in the database. Useful after registering new images with `image_registrator`.

    ```bash
    colmap point_triangulator \
      --database_path database.db \
      --image_path images/ \
      --input_path sparse/0 \
      --output_path sparse/0
    ```
  </Accordion>
  <Accordion title="image_registrator">
    Register new images against an existing model without running bundle adjustment or triangulation. Use this to extend a model after importing and matching new images, then follow up with `point_triangulator` and `bundle_adjuster`.

    ```bash
    colmap image_registrator \
      --database_path database.db \
      --input_path sparse/0 \
      --output_path sparse/0
    ```
  </Accordion>
</AccordionGroup>

### Dense reconstruction

<AccordionGroup>
  <Accordion title="image_undistorter">
    Undistort images and prepare a workspace for PatchMatch stereo or export to external dense reconstruction tools (CMVS/PMVS, CMP-MVS).

    ```bash
    colmap image_undistorter \
      --image_path images/ \
      --input_path sparse/0 \
      --output_path dense/ \
      --output_type COLMAP \
      --max_image_size 2000
    ```
  </Accordion>
  <Accordion title="patch_match_stereo">
    CUDA-accelerated dense stereo reconstruction. Computes depth and normal maps for all registered images. Requires the workspace to be prepared by `image_undistorter`.

    ```bash
    colmap patch_match_stereo \
      --workspace_path dense/ \
      --workspace_format COLMAP \
      --PatchMatchStereo.geom_consistency true
    ```
  </Accordion>
  <Accordion title="stereo_fusion">
    Fuse `patch_match_stereo` depth and normal maps into a dense colored point cloud (PLY format).

    ```bash
    colmap stereo_fusion \
      --workspace_path dense/ \
      --workspace_format COLMAP \
      --input_type geometric \
      --output_path dense/fused.ply
    ```
  </Accordion>
</AccordionGroup>

### Meshing

<AccordionGroup>
  <Accordion title="poisson_mesher">
    Reconstruct a surface mesh from the fused dense point cloud using screened Poisson surface reconstruction.

    ```bash
    colmap poisson_mesher \
      --input_path dense/fused.ply \
      --output_path dense/meshed-poisson.ply
    ```
  </Accordion>
  <Accordion title="delaunay_mesher">
    Reconstruct a surface mesh from a sparse or dense point cloud using a graph cut on the Delaunay triangulation with visibility voting. Accepts either a PLY file or a dense workspace folder.

    ```bash
    colmap delaunay_mesher \
      --input_path dense/ \
      --output_path dense/meshed-delaunay.ply
    ```
  </Accordion>
  <Accordion title="mesh_simplifier">
    Reduce mesh complexity using Quadric Error Metric (QEM) decimation while preserving shape and appearance.

    Key options:
    - `--MeshSimplification.target_face_ratio` — fraction of faces to retain (default `0.1`).
    - `--MeshSimplification.max_error` — maximum quadric error threshold (`0` = disabled).
    - `--MeshSimplification.boundary_weight` — boundary edge preservation weight (default `1000`).
    - `--MeshSimplification.num_threads` — threads for initialization.

    ```bash
    colmap mesh_simplifier \
      --input_path dense/meshed-poisson.ply \
      --output_path dense/meshed-poisson-simplified.ply \
      --MeshSimplification.target_face_ratio 0.25
    ```
  </Accordion>
  <Accordion title="mesh_texturer">
    Produce a texture atlas and UV coordinates for a triangle mesh using calibrated multi-view images from the undistorted workspace.

    ```bash
    colmap mesh_texturer \
      --workspace_path dense/ \
      --input_path dense/meshed-poisson.ply \
      --output_path dense/textured
    ```
  </Accordion>
</AccordionGroup>

### Model utilities

<AccordionGroup>
  <Accordion title="model_analyzer">
    Print statistics about a reconstruction: number of images, points, mean track length, mean reprojection error, etc.

    ```bash
    colmap model_analyzer \
      --path sparse/0
    ```
  </Accordion>
  <Accordion title="model_aligner">
    Geo-register or align a model to the coordinate system defined by given camera centers (e.g., GPS ground control points).

    ```bash
    colmap model_aligner \
      --input_path sparse/0 \
      --output_path sparse/0_aligned \
      --ref_images_path gps_reference.txt
    ```
  </Accordion>
  <Accordion title="model_converter">
    Convert COLMAP's binary or text export format to another format such as PLY or NVM.

    ```bash
    colmap model_converter \
      --input_path sparse/0 \
      --output_path sparse/0_ply \
      --output_type PLY
    ```
  </Accordion>
  <Accordion title="model_merger">
    Attempt to merge two disconnected reconstructions that share common registered images.

    ```bash
    colmap model_merger \
      --input_path1 sparse/0 \
      --input_path2 sparse/1 \
      --output_path sparse/merged
    ```
  </Accordion>
  <Accordion title="model_splitter">
    Divide a reconstruction into rectangular sub-models defined by a bounding box file, a maximum sub-model extent, or a number of subdivisions per dimension.

    ```bash
    colmap model_splitter \
      --input_path sparse/0 \
      --output_path sparse/split \
      --split_params 2,2,1
    ```
  </Accordion>
  <Accordion title="model_clusterer">
    Split a reconstruction into smaller sub-model clusters. Useful for managing and processing large-scale reconstructions.

    ```bash
    colmap model_clusterer \
      --input_path sparse/0 \
      --output_path sparse/clusters
    ```
  </Accordion>
  <Accordion title="model_comparer">
    Compare statistics between two reconstructions, such as camera pose differences and point cloud overlap.

    ```bash
    colmap model_comparer \
      --input_path1 sparse/0 \
      --input_path2 sparse/reference
    ```
  </Accordion>
  <Accordion title="model_cropper">
    Crop a reconstruction to a specific bounding box defined in GPS or model coordinates.

    ```bash
    colmap model_cropper \
      --input_path sparse/0 \
      --output_path sparse/cropped \
      --gps_min_lat 48.0 --gps_max_lat 48.1
    ```
  </Accordion>
  <Accordion title="model_transformer">
    Apply a similarity or rigid transform to the coordinate frame of a reconstruction.

    ```bash
    colmap model_transformer \
      --input_path sparse/0 \
      --output_path sparse/0_transformed \
      --transform_path transform.txt
    ```
  </Accordion>
  <Accordion title="model_orientation_aligner">
    Align the coordinate axes of a model using a Manhattan world assumption (aligns to dominant scene planes).

    ```bash
    colmap model_orientation_aligner \
      --input_path sparse/0 \
      --output_path sparse/0_aligned
    ```
  </Accordion>
</AccordionGroup>

### Database utilities

<AccordionGroup>
  <Accordion title="database_creator">
    Create an empty COLMAP SQLite database with the required schema.

    ```bash
    colmap database_creator \
      --database_path database.db
    ```
  </Accordion>
  <Accordion title="database_cleaner">
    Remove specific or all data tables from a database (features, matches, two-view geometries, etc.).

    ```bash
    colmap database_cleaner \
      --database_path database.db \
      --type all
    ```
  </Accordion>
  <Accordion title="database_merger">
    Merge two databases into a new database. Camera and image identifiers may change during the merge; cameras are not merged.

    ```bash
    colmap database_merger \
      --database_path1 database1.db \
      --database_path2 database2.db \
      --merged_database_path merged.db
    ```
  </Accordion>
</AccordionGroup>

### Vocabulary tree

<AccordionGroup>
  <Accordion title="vocab_tree_builder">
    Build a vocabulary tree from a database with extracted images. As a rule of thumb, use at least 10–100× more features than visual words. Pre-trained trees are available at https://demuc.de/colmap/.

    ```bash
    colmap vocab_tree_builder \
      --database_path database.db \
      --output_path vocab_tree.bin \
      --num_visual_words 65536
    ```
  </Accordion>
  <Accordion title="vocab_tree_retriever">
    Perform vocabulary tree based image retrieval to find visually similar images.

    ```bash
    colmap vocab_tree_retriever \
      --database_path database.db \
      --vocab_tree_path vocab_tree.bin \
      --output_index retrieval_index.bin
    ```
  </Accordion>
</AccordionGroup>

### Miscellaneous

<AccordionGroup>
  <Accordion title="automatic_reconstructor">
    Run the full sparse and dense reconstruction pipeline end-to-end for a set of input images. Tune quality, data type, feature algorithm, SfM mapper, and mesher method via options.

    ```bash
    colmap automatic_reconstructor \
      --workspace_path /path/to/project \
      --image_path /path/to/project/images \
      --quality HIGH \
      --mapper INCREMENTAL
    ```
  </Accordion>
  <Accordion title="bundle_adjuster">
    Run global bundle adjustment on a reconstructed scene. Useful to refine intrinsics or after running `image_registrator`.

    ```bash
    colmap bundle_adjuster \
      --input_path sparse/0 \
      --output_path sparse/0
    ```
  </Accordion>
  <Accordion title="color_extractor">
    Extract mean colors for all 3D points of a model from the images.

    ```bash
    colmap color_extractor \
      --image_path images/ \
      --input_path sparse/0 \
      --output_path sparse/0
    ```
  </Accordion>
  <Accordion title="point_filtering">
    Filter sparse 3D points by enforcing criteria such as minimum track length, maximum reprojection error, and minimum triangulation angle.

    ```bash
    colmap point_filtering \
      --input_path sparse/0 \
      --output_path sparse/0_filtered
    ```
  </Accordion>
  <Accordion title="image_filterer">
    Filter images from a sparse reconstruction based on registration quality criteria.

    ```bash
    colmap image_filterer \
      --input_path sparse/0 \
      --output_path sparse/0_filtered
    ```
  </Accordion>
  <Accordion title="image_deleter">
    Delete specific images from a sparse reconstruction by name.

    ```bash
    colmap image_deleter \
      --input_path sparse/0 \
      --output_path sparse/0_cleaned \
      --image_names_path images_to_delete.txt
    ```
  </Accordion>
  <Accordion title="image_rectifier">
    Stereo rectify cameras and undistort images for stereo disparity estimation.

    ```bash
    colmap image_rectifier \
      --image_path images/ \
      --input_path sparse/0 \
      --output_path rectified/
    ```
  </Accordion>
  <Accordion title="rig_configurator">
    Configure rigs and frames in the database after feature extraction. Use this for multi-sensor or multi-camera rig setups.

    ```bash
    colmap rig_configurator \
      --database_path database.db \
      --rig_config_path rig_config.json
    ```
  </Accordion>
  <Accordion title="rotation_averager">
    Run standalone rotation averaging on the view graph. Estimates global camera rotations from pairwise relative rotations.

    ```bash
    colmap rotation_averager \
      --database_path database.db \
      --input_path sparse/0 \
      --output_path sparse/0
    ```
  </Accordion>
  <Accordion title="view_graph_calibrator">
    Calibrate camera intrinsics using the view graph by estimating focal lengths and other parameters from pairwise geometric relations. Run before `global_mapper` when good prior intrinsics are not available. This command modifies the database in-place — work on a copy.

    ```bash
    colmap view_graph_calibrator \
      --database_path database_global.db
    ```
  </Accordion>
  <Accordion title="project_generator">
    Generate `.ini` project configuration files at different quality settings without running any reconstruction.

    ```bash
    colmap project_generator \
      --output_path project.ini \
      --quality HIGH
    ```
  </Accordion>
</AccordionGroup>

## Visualizing results

After running the CLI pipeline you can inspect results in the COLMAP GUI or with external tools:

- **Sparse point cloud**: In the GUI, choose **File > Import** and select the sparse model folder (`cameras.txt`, `images.txt`, `points3D.txt`).
- **Dense point cloud** (`fused.ply`): In the GUI, choose **File > Import from...** and select `fused.ply`.
- **Dense mesh** (`meshed-*.ply`): Use an external viewer such as Meshlab. Use `mesh_simplifier` to reduce mesh size before viewing, and `mesh_texturer` to add a texture atlas.
